# Lancers通知メールの実フォーマット(2026-06-05 実機確認)

## 確定事項

| 項目 | 値 |
|---|---|
| 送信元 | `noreply@lancers.co.jp`(表示名「ランサーズ事務局」)。**`lancers.jp` ではない** |
| 対象件名 | `[ランサーズ] 【新着】<カテゴリ群>カテゴリに、新しい仕事が登録されました!` |
| 頻度 | ほぼ毎日 1〜2通(JST 10時台に多い) |
| 推奨クエリ | `from:noreply@lancers.co.jp newer_than:7d` |

## 本文構造(text/plain)

```
Ink6x さん
...
━━━━━━━━━━━━━━━━━━━━━━━
■■　<サブカテゴリ名>　■■ （N件）
━━━━━━━━━━━━━━━━━━━━━━━
--------------------------------------------------------------------
▼ <案件タイトル>
[依頼金額] 100,000円 ～ 200,000円
[方式] プロジェクト
[募集締切] 2026年6月6日 18:14
https://www.lancers.jp/work/monitor/<案件ID>/new_work_mail/　← 末尾に全角スペースあり
```

- 1通に複数サブカテゴリ(Webシステム開発、サーバー、PM、保守運用、AI自動化など)
- 案件URLは `/work/monitor/<id>/new_work_mail/`(トラッキング用)。正規URLは `/work/detail/<id>`
- 末尾にカテゴリ検索ページのURL(`/work/search/...`)があるが案件ではないので除外する

## その他のメール種別(パーサーはフォールバックで対応)

| 件名 | 扱い |
|---|---|
| `仕事の招待状が届きました` | フォールバックで案件URL抽出(クライアントからの直接招待。優先度高いので将来専用対応の価値あり) |
| `新しい端末からのログインを検知しました` | 案件なし(0件でスルー) |
| `magazine@lancers.co.jp` のメルマガ | クエリで除外済み |
| `lancers_XXXXさんからのメッセージ受信通知` | 案件なし。将来「返信あり」検知に使える可能性 |
